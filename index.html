下面是说明，请安心食用~

按钮说明
- 左上角 < 是返回重新上传文件，会同时清除本地浏览器在 indexedDB 存储的 conversations.json 所有数据。
- 右上角 :D 可切换亮色与暗色模式。

数据总览
- 消息是根据 current_node 进行分析回溯的，不会收录重新编辑或生成的消息。
- 并将 " Model set context updated. " 等系统消息合并入 Agent 消息。
- 所以双方的消息数是相同的。
- conversations.json 数据是用 indexedDB 存在本地浏览器里的， 点击返回按钮 < 自动清除，安全可靠的哈。
趋势图表
- 显示最近有对话的 20 天的消息数量变化，没有对话的日期会直接跳过。
- 点击趋势图的柱形，会显示日期和当天消息数哦。

收藏功能
- 点击对话列表右侧的圆形即可收藏对话。
- 由于重新上传文件时 indexedDB 就会清空，所以单独的收藏功能及其没用， 于是做成可以多选对话导出。

搜索排序
- 搜索框目前只支持根据标题找对话。
- 可按时间、消息数或标题首字排序。

导出数据
- 导出的数据是干净的 MarkDown 格式，很方便看。
